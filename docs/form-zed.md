# formZed Utility Function

The `formZed` function provides predefined field configurations for commonly used form inputs, such as name, email, phone number, and password. It helps maintain consistency in form validation and input attributes.

## üìå Features

‚Ä¢ Returns input field configurations, including `placeholder`, `type`, `maxLength`, and `minLength`.  
‚Ä¢ Standardizes form input attributes for common fields.  
‚Ä¢ Supports various input types, such as `text`, `email`, `password`, and `tel`.  
‚Ä¢ Automatically generates a title by formatting the field name.  
‚Ä¢ Ensures validation constraints are met.

```bash
pnpm add @estarlincito/utils

```

## ‚ö° Usage

The `formZed` function simplifies the process of configuring form fields with predefined values.

### Example 1: Get Configuration for an Email Input

```ts
const emailField = formZed('email');
console.log(emailField);
/* 
{
  placeholder: 'name@domain.com',
  type: 'email',
  maxLength: 30,
  minLength: 5,
  title: 'Email'
}
*/
```

### Example 2: Get Configuration for a Password Input

```ts
const passwordField = formZed('password');
console.log(passwordField);
/* 
{
  placeholder: '4ppl3-1Ph0ne',
  type: 'password',
  maxLength: 30,
  minLength: 8,
  title: 'Password'
}
*/
```

## ‚ùå Incorrect Usage

```ts
const field = formZed('address'); // ‚ùå Returns undefined as 'address' is not defined in formZed
```

## ‚úÖ Correct Usage

```ts
const field = formZed('username'); // ‚úÖ Returns configuration for 'username'
```

## üõ† How It Works

‚Ä¢ Accepts a field name and looks up its predefined configuration.  
‚Ä¢ If the field is found, it returns an object containing its attributes.  
‚Ä¢ The field title is automatically generated by capitalizing the first letter and replacing hyphens with spaces.  
‚Ä¢ If the field is not found, it logs an error using `handleError` and returns `undefined`.

## ‚ö†Ô∏è Error Handling

If an invalid field name is provided, `formZed` calls `handleError` and logs an error message:

```ts
handleError(`${name} was not found in (fiels) /lib/zed.ts`);
```

This ensures better debugging and prevents unexpected behavior.
